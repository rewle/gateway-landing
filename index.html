Приложение B · Решение

Один шлюз на границе двух компаний.

Gateway устанавливается в контуре каждого партнёра. Ни одна из сторон не получает прямого доступа к источнику данных другой — только то, что явно разрешено правилом.

Ф

Фильтрует

Передаются только поля, перечисленные в правиле маршрута — остальное отбрасывается до выхода из контура.

М

Маскирует

Персональные данные заменяются хешем или скрываются, если партнёру нужен факт, а не значение.

Л

Логирует

Каждый запрос фиксируется — кто, что, кому и когда передал. Готовый журнал для аудита и регулятора.

Приложение C · Порядок действий

Как проходит один запрос

  1. 01

    Партнёр А задаёт правило

    Какие поля, какому потребителю, на каких условиях — маршрут настраивается в панели, без строчки кода.

  2. 02

    Партнёр Б обращается через API

    Запрос проходит по токену или подписанному secure-каналу между двумя шлюзами — без прямого доступа к источнику.

  3. 03

    Gateway маскирует поля в реальном времени

    Лишние данные не покидают контур; персональные — превращаются в хеш прямо в потоке ответа.

  4. 04

    Запрос попадает в журнал

    Кто получил, что именно и когда — записано и доступно обеим сторонам как доказательство при проверке.

Приложение D · Возможности

Фильтрация и хеширование полей

Правила вплоть до вложенных путей JSON — allow, mask или remove на каждое поле маршрута.

Контроль доступа

Токен на потребителя, allow-листы по IP на источник и назначение для исходящих маршрутов.

Двусторонний безопасный обмен

Подписанный канал между узлами: запрос и хеш тела заверяются ключом, срок жизни подписи — 60 секунд.

Забор данных по расписанию

Плановые задания сами обращаются к источнику и пересылают результат через нужный маршрут.

Безопасный канал между узлами

Взаимная аутентификация шлюзов через подписанный discovery — доверенные пиры, без ручных сертификатов на каждую пару.

Полный журнал операций

Каждая передача — с полем, статусом и получателем — сохраняется отдельно от рабочей базы, для аудита без нагрузки на прод.

Приложение E · Альтернативы, которые уже пробовали

Написать самим

6–18 месяцев разработки, своя команда поддержки, ни одной готовой сертификации на старте.

Интеграционная шина (ESB)

Решает общую задачу интеграции, но не заточена под маскирование полей и обмен между разными юрлицами.

DLP / SIEM

Защищает периметр внутри компании — не решает задачу совместного использования данных с партнёром.
